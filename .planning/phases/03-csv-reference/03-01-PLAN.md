# ASSUMPTION: CSV Column Specification
Based on typical shipping rate table patterns (Shopify shipping zones, weight-based rates, price-based rates), this plan assumes the following columns. If the actual MatrixRates app uses different column names, this plan should be revised:
- zone_name (string, required) — shipping zone identifier
- min_weight (number, required) — minimum weight in grams
- max_weight (number, required) — maximum weight in grams  
- shipping_price (number, required) — price in store currency
- condition (string, optional) — "price", "weight", or "flat" (default: "weight")
- currency (string, optional) — three-letter currency code (default: USD)

This assumption is based on:
- Common Shopify shipping app patterns
- Standard weight-based shipping zone models
- Explicit blocker in CLAUDE.md: "Phase 3: Exact CSV column specification from the app"
